## 前提となる実測事実(詳細: eval/REPORT.md「委譲検証」)

- 委譲のオーケストレーション費は **~$0.11〜0.15 の固定 floor**(呼び出し元 Claude Code 側の構造費、委譲先非依存 — Haiku 対照実験の H1 で確認)。40ファイル規模の重量級タスクでも floor は不変($0.129、第4ラウンド課題4)。損益分岐は baseline ≈ $0.13〜0.15。

- **委譲の現実的な節約上限は −30〜50%帯**(第4ラウンド課題4の反証)。「重量級ほど委譲が効く(baseline $1 超で −80% 級)」という外挿は不成立: baseline はファイル数に線形でスケールせず、機械的で規則を記述できるタスクは Sonnet がスクリプトに畳んで償却する(「ローカル27Bに委譲できるほど機械的なタスクは、Sonnet がスクリプト化できるほど機械的でもある」という構造的天井)。
