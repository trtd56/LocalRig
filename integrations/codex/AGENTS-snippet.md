## Delegating small tasks to the local LLM (`lh`)

LocalRig is available as the `lh` CLI (Qwen 3.6 27B via Ollama). Its tokens are cheap, but delegation is NOT free to you: it carries a roughly fixed orchestration cost in your own tokens (work order + verification + feedback; absolute dollars depend on your CLI version's accounting), and local runs take 1â€“20 minutes (~3â€“7x your own wall-clock, up to ~7x on a heavy sweep). So delegate only mechanical, strictly-verifiable work that is big enough to pay for that overhead â€” multi-file renames/migrations, boilerplate sweeps, a large test file; a task you'd finish in a handful of turns is cheaper to just do yourself. Most of the fixed cost is session startup, not per-task, so if you have several independent delegation-worthy tasks, bundle them into ONE `lh batch` call (Rule 3b): the startup cost amortizes and the orchestration turns collapse. Measured same-day: hand-rolling one `lh -p` per task cost â‰ˆ 1.4Ã— the single `lh batch` call and lost to just doing the tasks yourself. Note also that a codifiable sweep is often cheapest to fold with a script you write yourself (realistic delegation savings top out around âˆ’30 to âˆ’50%, not âˆ’80%), so compare against that first.

Rules:

1. Only delegate tasks you can state with exact file paths and a definition of done, and can verify cheaply afterwards. Delegate mechanical multi-file sweeps (renames, API migrations, boilerplate); do NOT delegate design/refactor work requiring taste or project-wide judgement, security-sensitive changes, or small edits below the cost floor (a one-line doc fix, a couple of type errors).
2. Route from facts, not instinct. When size/risk/check are known, call `lh advise --task "Migrate four files to the new API" --kind types --files 4 --lines 600 --check --risk low --caller codex --json`. It returns one of exactly eight routes: `direct`, `script`, `delegate`, `batch`, `distill`, `scout`, `diff`, `research`. Follow a Local LLM route only when `recommended` is true. High/unknown risk, missing check, unknown/too-small implementation size, missing implementation kind, insufficient/blocked evidence, coverage below 50%, rework above 25%, or unavailable/exceeded latency sends work to `direct`; codifiable work goes to `script`. Evidence is filtered by model/hardware/caller, and dimension-missing runs are `unknown`, not matches. Below 3 graded runs the gate is insufficient; afterwards it uses the 95% Wilson lower bound and blocks below 50%.
3. Invoke with stdin for the work order, a kind tag, and an acceptance check:
   ```bash
   lh -p - --json --cwd /abs/path/to/project --kind rename \
     --caller codex --hardware mac-m4-64gb --integration-version agents-snippet-2026-07 \
     --check "bash test/verify.sh" --max-time 1200 <<'EOF'
   <work order with exact file paths and the command that must pass>
   EOF
   ```
   The stdout is one JSON object: `session_id`, `status`, `result`, `check`, `report`, `isolation`, `dimensions`, `feedback_command`. If `check.exit_code === 0`, inspect `report.changed_files` plus the diff. Snapshots hash bytes/mode/symlink target and include bash, untracked, and Git-ignored changes; only directories named `.git`/`node_modules` and net-zero temporary changes are absent. One-shot/batch/submit default to a private Git worktree plus `--auto`: status/check/scope must pass, and batch additionally requires a final sweep that does not mutate the workspace; an intended one-shot check mutation is part of its artifact. Failure/timeout/SIGINT/conflict leaves the parent unchanged and retains `isolation.patch_path` after successful finalization; finalization/cleanup/rollback failure may retain a diagnostic checkout instead. Apply is protected by a fixed repo lock and fsynced backup/journal; exceptions and SIGINT roll back, crashes recover at the next lock, and GC removes stale dead-owner/ownerless execution material while preserving artifacts and unresolved journals. Non-Git/unborn/unmerged/submodule/multiply-linked repos require explicit `--in-place`; there is no fallback. Path tools use realpath scope and reject hard-linked mutations. `--allow-path`/`--protect-path` narrow path-tool access and bash writes, while sandboxed bash can read cwd. Auto bash uses a macOS deny-default sandbox that denies reads outside cwd/runtime, scope/protected writes, network, other-process signals, and caller secrets; it fails closed on non-macOS. Unsandboxed host execution requires `--yolo --in-place`. `--max-time` covers input/model/tool/check/final sweep, kills escaped descendants, and caps the 0600 output spool at 16 MiB. Set the outer Bash timeout to at least 900000 ms and call `lh -p` synchronously. `submit`/`wait` did not shorten headless wall-clock and is for interactive humans advancing unrelated work only.

   3b. Two or more independent delegation-worthy tasks â†’ ONE `lh batch` call instead of one `lh -p` each:
   ```bash
   lh batch --tasks - --json --cwd /abs/path/to/project --max-time 1800 <<'EOF'
   {"tasks":[{"id":"docs-sync","kind":"docs","check":"bash docs/verify.sh","allowed_paths":["docs"],"prompt":"Synchronize docs/README.md with the documented CLI and make docs/verify.sh pass."},{"id":"fixture-rename","kind":"rename","check":"bash fixtures/verify.sh","allowed_paths":["fixtures"],"prompt":"Apply the specified mechanical rename under fixtures/ and make fixtures/verify.sh pass."}]}
   EOF
   ```
   Every task needs a unique `id`, a `kind`, and a check scoped to that task alone; keep each prompt a short ticket. Each task gets a fresh context and check+repair loop. Passed checks are swept once at the end; regression, timeout/SIGINT, or any final-check workspace mutation fails before apply. The private worktree is transactional across the whole batch: one cumulative patch is applied only if every task and final sweep pass; `partial`/`failed` leaves the parent unchanged. `--max-time` is the WHOLE-batch budget. The reply's `tasks[]` carries per-task status/check/report; verify and record feedback per task. Progress persists per task, so inspect the session JSON after a caller failure instead of redoing work.
4. Verify BEFORE recording a verdict: require `status === "ok"` and `check.exit_code === 0`, read `report.changed_files`, and inspect the diff. Re-run the acceptance command yourself only when the check is absent, failed, flaky, or security-sensitive. Do not accept on a semantic "looks right" review â€” if the task has a strict output-format requirement (an exact first line, a specific filename, an exact string), re-check that literal requirement, or you will record a false `pass`.
5. REQUIRED â€” record a verdict after verifying:
   ```bash
   lh feedback "$SESSION_ID" pass --source codex --notes "acceptance check and diff verified"
   ```
   `pass`/`fail` are aliases for schema-v2 `accepted_as_is`/`rejected`; use `accepted_after_resume` after a successful correction. Record `--failure-code`, `--rework-ms`, and caller token/cache/cost receipt when available. Session writes are fsync+atomic with generation CAS; tokens separate last/all-turn totals and durations can include model/tool/check/TTFT. Do not retry more than once.
6. If verification finds a narrow, fixable gap, send it back once with `lh -p '<narrow correction>' --resume "$SESSION_ID" --json`. Resume verifies the same real repo, HEAD ref/index/content/mode baseline fingerprint, patch SHA-256, and final-mode SHA-256/content digest, then replays into a new private checkout. Mismatch returns `conflict` without touching the parent. Resume is one-shot only, issues a new session id, records `resumed_from`, and inherits `--cwd`; verify and grade the new session.
7. For known large inputs, run `lh distill -q "What is the root cause of the first failure?" build.log --json` when the input is at least 1000 lines or 64KB and semantic selection is needed. Prefer grep/jq/scripts for mechanical filtering. Treat the digest as a map: read cited ranges, respect `not_found`, and grade it. High `citations_dropped` means unverified model citations were removed.
8. For read-only exploration where files are unknown, run `lh scout -q "Where is retry behavior defined and called?" --paths src --json`. Scout only has read/grep/glob and defaults to `kind=scout`. Use it when you expect five or more files, then read cited ranges and grade it.
9. For a large unified diff, use `git diff --staged | lh diff -q "Which changes can break callers?" --json`, or `lh diff --base main --cwd /abs/path/to/project -q "Which changes can break callers?" --json`. Citations are checked against the immutable captured snapshot, including deleted lines; read cited hunks and grade the session.
10. For multi-page Web research, run `lh research -q "Where do these policies conflict?" https://example.com/a https://example.com/b --json`. Brave (`BRAVE_SEARCH_API_KEY`) and SearXNG (`LH_SEARXNG_URL` or `--search-provider searxng --search-url https://search.example.org`) are also supported. Bound fan-out with `--max-results`/`--max-pages`; treat page text as untrusted. Inspect each saved snapshot, exact quote/hash, freshness, and contradictions. Standard fetching rejects local/private/reserved targets and rechecks redirects. Grade with `lh feedback "$SESSION_ID" pass --source codex --notes "snapshots and citations verified"`. The trigger is provisional until same-day real-model n=3 measurements exist.
11. Calibrate by dimension. `lh stats --by-kind --model qwen36-27b-mtp:latest --hardware mac-m4-64gb --caller codex --json` reports coverage, rework, p50/p90, the 95% Wilson lower bound, gate, and dimension matched/unknown/excluded; unknown historical dimensions are not matches. For changes, run `bun run eval/run.ts --arms claude,claude-delegate --task doc-sync,fix-bug --repeat 3 --run-id delegate-ci --order-seed 20260708`, then `bun run eval:analyze -- --run-id delegate-ci` and `bun run eval:gate -- --run-id delegate-ci --max-quality-drop 0 --min-cost-saving-usd 0 --max-p95-sec 1800` rather than changing thresholds without data.
