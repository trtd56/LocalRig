# バグ報告: parseRange("5-3") が降順レンジで空配列を返す

## 概要
`src/range.ts` の `parseRange` に降順レンジ(開始値 > 終了値)を渡すと、JSDoc に
書かれている `[5, 4, 3]` ではなく **空配列 `[]`** が返ってきます。昇順レンジ
(`"1-4"`)は正しく動くため、降順の分岐だけが壊れているように見えます。

本番のページネーション処理で、ユーザーが結果を「新しいページから古いページへ」
降順で範囲指定したときに一覧が丸ごと空になる不具合として発覚しました。

## 環境
- Bun 1.2.21
- macOS 14 (Darwin 23.6.0)
- 対象: `src/range.ts`(現行 main、コミット `d6f1060`)

## 再現手順
リポジトリ直下に次の `repro.ts` を置いて `bun run repro.ts` を実行します。

```ts
import { parseRange } from "./src/range";

console.log("asc :", parseRange("1-4")); // 期待 [1,2,3,4]
console.log("desc:", parseRange("5-3")); // 期待 [5,4,3]
```

実行すると、こちらの環境では次のようになりました。

```
asc : [ 1, 2, 3, 4 ]
desc: []
```

昇順は仕様どおりですが、降順(`"5-3"`)だけ空配列が返っています。

## 期待する挙動
`parseRange("5-3")` は JSDoc の仕様どおり `[5, 4, 3]` を返すべきです。

## 影響と再現性
- ページネーションの「5ページ目から3ページ目まで」の降順表示が常に空になります。
- 手元では毎回再現します。優先度: 高。

## 補足
おそらく `from <= to` を前提にしたループ条件が降順のときに一度も回らず、結果が
空のまま返っているのではないかと推測しています(未確認)。修正をお願いします。
