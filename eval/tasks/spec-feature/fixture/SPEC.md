# 検索クエリ言語 仕様

タスク一覧に対する検索関数 `search(tasks, query)` が受け取る、クエリ文字列の言語仕様を定義する。
`src/search.ts` の `search()` は現在未実装(throw する)なので、本仕様に従って実装すること。

## 全体像

クエリは空白(スペース/タブ)区切りの **トークン列**。各トークンは 1 つの **フィルタ** を表す。
タスクは、クエリ中の **すべてのフィルタを満たす(AND)** ときにのみ検索結果に含まれる。
結果は入力の `tasks` と **同じ順序** を保つ。

空クエリ(空文字列、または空白のみ)はフィルタが 0 個なので **全タスクをそのまま返す**。

## トークンの種類

### status フィルタ: `status:open` / `status:done`
`task.status` が指定値に一致するタスクだけを通す。
値は `open` または `done` のみ。それ以外(例 `status:archived`)は **Error を投げる**。

### tag フィルタ: `tag:xxx`
`task.tags` に `xxx` を **完全一致** で含むタスクだけを通す。タグ比較は **大文字小文字を区別する**
(`tag:code` と `tag:CODE` は別物として扱う)。複数指定はそれぞれ独立したフィルタとなり、AND で
絞り込まれる(例 `tag:code tag:urgent` は両方のタグを持つタスク)。

### 除外(否定): 先頭 `-`
トークンの直前(空白を挟まず)に `-` を付けると、そのフィルタの意味が反転する。
- `-tag:urgent` … `urgent` タグを **持たない** タスクだけを通す
- `-status:done` … `done` **でない** タスクだけを通す
- `-foo` … タイトルに `foo` を **含まない** タスクだけを通す

### 裸の単語: `foo`
`task.title` に `foo` を **部分一致** で含むタスクを通す。比較は **大文字小文字を無視** する。
単語を複数並べた場合はそれぞれが独立したフィルタとなり、**すべてを含む(AND)** タスクだけが残る
(例 `write notes` は title に `write` と `notes` の両方を含むもの)。

### 引用符フレーズ: `"quoted phrase"`
ダブルクォートで囲んだ部分は、内部の空白ごと **1 つのフレーズ** として扱い、title にその並びを
部分一致(大文字小文字を無視)で含むタスクを通す。
`"write doc"` は title に連続した `write doc` を含むものにだけ一致する(`write` と `doc` が
離れて存在するだけでは一致しない)。

引用符で囲んだトークンは **常にテキスト(フレーズ)として扱う**。中に `:` があってもキー付き
フィルタとは解釈しない(例 `"status:open"` は「title に文字列 `status:open` を含む」の意味)。
否定 `-` はクォートの外側に付ける(例 `-"quoted phrase"`)。

## エラー条件

- 未知のキー(`status` / `tag` 以外)を使った場合。例 `foo:bar`、`priority:high` は **Error**。
- `status:` の値が `open` / `done` 以外の場合。例 `status:archived` は **Error**。
- ダブルクォートが閉じていない場合。例 `"unterminated` は **Error**。

## 実装の分割

- `src/query/parse.ts` … クエリ文字列を構造化フィルタへ変換する(パーサ)。
- `src/query/apply.ts` … 構造化フィルタをタスク列に適用して絞り込む。
- `src/search.ts` の `search()` … 上記 2 つを使って `search(tasks, query)` を実装し直す。

内部のデータ構造や関数名は任意でよいが、`search()` は必ず `src/query/parse.ts` と
`src/query/apply.ts` を import して使うこと。
